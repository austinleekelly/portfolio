**About the Role:**

We’re looking for someone who can do two important things:

- Help us solve tough product support questions that go beyond Tier 1 troubleshooting- Build clear, user-friendly documentation for our product — including brand-new features and existing ones that don’t have anything written yet.

This is a hands-on role for someone who enjoys digging into logs and code (Java), figuring out what’s going wrong, and explaining it in plain terms. You won’t be writing the code to fix it, but you _will_ need to help pinpoint the problem and give our engineering team a strong head start. You’ll also need to move quickly — we work in a fast-paced environment with live client issues, and being able to respond swiftly and clearly is key.

  

**What You'll Do:**

- Investigate product issues that Tier 1 support can’t solve- Read logs and Java code to help identify bugs or unexpected behavior- Triage and resolve incoming tickets quickly — especially when they involve live client issues- Write clear, friendly documentation for product features — both new and existing- Work closely with product and engineering to understand how things work- Suggest process improvements or common support themes we should address- Maintain and improve our knowledge base

  

**What We're Looking For:**

- Experience supporting a technical SaaS product- Comfortable reading code (Java preferred) and navigating logs- Excellent writing skills — you can explain complex ideas simply- Self-starter who’s excited to build docs from the ground up- Organized, curious, and detail-oriented- Calm under pressure and responsive in time-sensitive situations

  

**Bonus Points If You Have:**

- Experience in QA, technical writing, or developer support- Worked with tools like Confluence, Zendesk Knowledge Base, or Git-based docs- Written internal runbooks or playbooks from 0-1

  

**Location:**

This is a remote role. We work async-friendly and value clear communication.
